Session 01 — Claude Managed Agents

Every agent is built from four primitives

Identity
Agent
System prompt, model, skills, tools, and permissions — the agent's stable identity
Template
Environment
Network allow list and pre-installed packages — the sandbox the agent runs in
Runtime
Session
Provisions the harness, mounts credentials, and starts Claude — the live execution unit
Output
Events
Durable structured transcripts of every action, tool call, and lifecycle transition

Session 01 — Now on Claude Platform on AWS

Memory, Vaults,
and multi-agent

01
Memory + Dreaming — agents persist context across sessions; Claude automatically consolidates learnings into curated memory after each run
02
Vaults — Claude sees a placeholder token; the real credential is injected at the network layer. Claude never touches the secret — the compliance answer for enterprise agentic deployments
03
Multi-agent orchestration — Claude delegates to subagents with independent context windows for true parallel execution; now available on the Claude Platform on AWS

Session 02 — Three levels of autonomy

Verification,
multi-tasking, background

01
Verification loops — spin up server, give Claude browser access, define "done" with screenshots or log checks; distill into a skill file anyone can reuse
02
Multi-tasking — Claude Code desktop and claude agents CLI manage parallel sessions with attention indicators
03
Background loops — /loop runs a prompt on a recurring schedule; Routines extend this to the cloud, triggered by time or events

Session 02 — Task Budgets

Turn budgets
into quality

01
Set a token budget — let Claude decide how to use it most effectively, rather than hardcoding explicit rules
02
Higher budget → more thorough exploration, richer output. Lower budget → faster, leaner result. Same prompt, tunable quality
03
Available now with Claude Opus 4.8 in the managed agents harness — prefer task budgets over hardcoded step counts

Session 02 — Claude 4 / Interleaved Thinking

Reasoning
between tool calls

01
Previously: Claude reasoned once, then called all tools. Now: reasoning interleaves with each tool call — Claude adapts its plan as results come in
02
More adaptive agentic behavior — each tool result shapes the next reasoning step, producing better decisions in multi-step tasks
03
Directly relevant for Bedrock Agents and Claude Code workflows that chain multiple tools — improved reasoning quality without prompt changes

What connected the sessions

Key themes

Managed infrastructure is here

Claude Managed Agents handles hosting, scaling, sessions, Memory, and Vaults — available on the Claude Platform on AWS. Developers focus on agent design, not DevOps

Cost and quality are now tunable

Task Budgets let teams dial quality vs. cost without rewriting prompts — set a token budget, let Opus 4.8 decide how to use it. Same workflow, different fidelity

Trust is built through verification

Verification loops, interleaved reasoning, and structured event output all serve one goal: earning enough confidence in agent behavior to remove the human from the loop

Japan is already building

Packed room, deep technical questions, AWS SA on stage. Japanese developers — Rakuten included — are past evaluation and scheduling production agent runs
