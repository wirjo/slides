Session 01 — Claude Managed Agents

Every agent is built from four primitives

Identity
Agent
System prompt, model, skills, tools, and permissions — the agent's stable identity
Template
Environment
Network allow list and pre-installed packages — the sandbox the agent runs in
Runtime
Session
Provisions the harness, mounts credentials, and starts Claude — the live execution unit
Output
Events
Durable structured transcripts of every user action, agent turn, tool call, and lifecycle transition

Session 01 — Enterprise features

Memory, Vaults,
and multi-agent

01
Memory + Dreaming — agents persist context across sessions and automatically consolidate learnings into curated memory after each run
02
Vaults (secure env vars) — Claude sees a placeholder token; real credentials are injected at the network layer. Claude never touches the actual secret — clean compliance story
03
Multi-agent orchestration — Claude delegates to subagents with independent context windows for true parallel execution
04
Scheduled deployments — recurring session triggers for reports, monitoring, and any repeating workflow
05 / 11
Session 02 — Three levels of autonomy

Verification,
multi-tasking, background

01
Verification loops — spin up a server, give Claude browser access, define "done" with screenshots or log checks. Distill the pattern into a skill file anyone can reuse
02
Multi-tasking — Claude Code desktop and CLI both manage parallel sessions with attention indicators. claude agents in the terminal gives the same session view as the sidebar
03
Background loops — /loop runs a prompt on a recurring schedule (e.g., babysit PRs, fix CI every 10 min). Routines extend this to the cloud, triggered by time or events

What connected the sessions

Key themes

Infrastructure is solved

Managed Agents abstracts away hosting, scaling, sessions, and sandboxing. The conversation is now about agent design — not DevOps

Trust is the unlock

Verification loops, Vaults, and structured event output all serve one goal: earning enough trust in agent behavior to let it run unsupervised

Enterprise is ready

Rakuten is already scheduling agent runs for reports and monitoring — with Vaults handling the governance requirements that previously blocked deployment

Japan is building

Deep questions, packed room, AWS SA already presenting. Japanese developers are past evaluation — they want architecture and production patterns
