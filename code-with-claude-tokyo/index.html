Task Budgets · Reasoning effort

Tunable quality,
tunable cost

01
Task Budgets — set a token budget; let Opus 4.8 decide how to allocate it across dynamic workflows. Higher budget triggers deeper exploration, more reasoning steps, richer output
02
Reasoning effort — dial thinking depth from low to high without rewriting prompts. Same task, same model, different fidelity — prefer this over hardcoded step counts or explicit rules
03
Both are designed to scale with model capability — as Claude improves, the same budget produces better results without any changes to your workflow
07 / 13
Claude 4 — Thinking modes

Three modes of thinking, each for a different depth of work

Mode 01
Adaptive thinking

Claude decides whether to think based on task complexity — simple tasks respond immediately; harder tasks trigger internal reasoning automatically. No explicit mode toggle needed

Best for: general-purpose use where you want Claude to self-regulate thinking cost

Mode 02
Extended thinking

A single deep reasoning block before the final answer — you set the budget, Claude uses it fully. Higher budget = more thorough exploration before committing to an answer

Best for: complex single-shot tasks — math, coding, analysis — where depth matters more than speed

Mode 03 — New in Claude 4
Interleaved thinking

Previously: Claude reasoned once, then called all tools. Now: Claude reasons between each tool call — adapting its plan as results come in, not committing to a full plan upfront

Best for: agentic workflows with multiple tool calls where intermediate results should change the next step
